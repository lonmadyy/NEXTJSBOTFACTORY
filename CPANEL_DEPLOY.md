# Деплой Next.js проекта на cPanel

Этот документ написан под текущий проект `BOT FACTORY` и под обычный сценарий деплоя на `cPanel -> Setup Node.js App` / `Application Manager`.

## Короткий вывод по проекту

На дату проверки `2026-04-04` проект **готов к деплою на cPanel**, но только при одном важном условии:

- ваш хостинг должен поддерживать **Node.js приложения**, а не только обычный PHP/статический `public_html` хостинг

### Что проверено и уже работает

- `npm run build` проходит успешно
- `npm run lint` проходит без ошибок
- `npx tsc --noEmit` проходит без ошибок
- запуск через `node app.js` проходит успешно
- открываются ключевые маршруты:
  - `/`
  - `/services`
  - `/robots.txt`
  - `/opengraph-image`

### Что уже подготовлено в проекте

- есть `app.js` для запуска через Passenger/cPanel
- в `package.json` есть `start:cpanel`
- в `package.json` указано требование по Node.js: `>=20.9.0`
- проект не требует обязательных `.env` переменных для базового запуска

### Важные ограничения

- Если на тарифе **нет Node.js Application Manager / Setup Node.js App / Passenger**, этот проект на таком cPanel нормально не поднять.
- Если на сервере Node.js ниже `20.9.0`, деплой может не заработать.
- Этот проект нужно деплоить как **Node.js приложение**, а не как просто набор файлов в `public_html`.
- В проекте используются `next/font/google` шрифты. Это значит, что во время `npm run build` сервер должен иметь доступ в интернет, чтобы скачать шрифты.
- В проекте есть динамические Open Graph / Twitter image маршруты. После деплоя их нужно отдельно проверить.

## Что именно деплоим

Это не статический HTML-сайт. Это `Next.js 14` приложение.

Поэтому правильная схема такая:

1. Загрузить исходники проекта на сервер.
2. Установить зависимости на сервере.
3. Собрать проект на сервере.
4. Создать Node.js приложение в cPanel.
5. Указать стартовый файл `app.js`.
6. Перезапустить приложение.

---

## Что должно быть на хостинге

Перед началом проверь в cPanel, есть ли у тебя:

- `Setup Node.js App` или `Application Manager`
- `Terminal`
- возможность выбрать Node.js `20.x` или `22.x`

Если этого нет, дальше можно не продолжать. Нужно либо:

- писать в поддержку хостинга и просить включить Node.js
- либо переносить проект на VPS/VDS
- либо деплоить на платформу вроде Vercel

---

## Какой способ деплоя лучше

Для новичка рекомендую такой порядок:

1. Локально проверить проект.
2. Загрузить исходники на сервер.
3. На сервере выполнить `npm ci` и `npm run build`.
4. Подключить проект через `Setup Node.js App`.

Это надёжнее, чем собирать всё на Windows и потом пытаться перенести готовую сборку на Linux.

## Что загрузить на сервер

На сервер нужно загружать **исходники проекта**, но не весь локальный мусор.

### Загружать нужно

- `app/`
- `components/`
- `lib/`
- `public/`
- `scripts/`
- `app.js`
- `package.json`
- `package-lock.json`
- `next.config.js`
- `tsconfig.json`
- `postcss.config.js`
- `tailwind.config.ts`
- `next-env.d.ts`

### Не нужно загружать

- `node_modules/`
- `.next/`
- `out/`
- `output/`
- `.git/`
- `.claude/`
- `.playwright-cli/`
- локальные `.zip` архивы
- временные файлы и логи

Если загружаешь вручную через архив, не клади в архив `node_modules` и `.next`.

---

## Куда загружать проект

Не загружай проект в `public_html`.

Лучше использовать отдельную папку, например:

```bash
/home/USERNAME/apps/botfactory
```

Или:

```bash
~/apps/botfactory
```

`public_html` нужен для обычных сайтов. Node.js приложение на cPanel обычно живёт отдельно.

## Самый понятный пошаговый деплой

Ниже сценарий для человека, который делает это первый раз.

## Шаг 1. Проверь проект локально

В корне проекта выполни:

```bash
npm run lint
npx tsc --noEmit
npm run build
```

Для текущего проекта эти проверки уже пройдены успешно.

## Шаг 2. Подготовь архив с проектом

Самый простой путь:

1. Создай папку-копию проекта для деплоя.
2. Удали из неё:
   - `node_modules`
   - `.next`
   - `out`
   - `output`
   - `.git`
   - локальные архивы
3. Запакуй оставшееся в `.zip`.

Если работаешь через Git, архив не нужен.

## Шаг 3. Загрузи проект на сервер

Есть 3 нормальных варианта:

- `File Manager` в cPanel
- `SFTP`
- `Git Version Control` в cPanel

Если ты новичок, чаще всего проще:

1. открыть `File Manager`
2. зайти в `/home/USERNAME/apps/`
3. создать папку `botfactory`
4. загрузить туда архив
5. распаковать его

## Шаг 4. Создай Node.js приложение в cPanel

Открой:

- `Setup Node.js App`
- или `Application Manager`

Создай приложение с такими параметрами:

- `Node.js version`: `20.x` или выше
- `Application mode`: `Production`
- `Application root`: `apps/botfactory`
- `Application URL`: твой домен или поддомен
- `Application startup file`: `app.js`

Если поле называется чуть иначе, смысл тот же: стартовый файл должен быть именно `app.js`.

## Шаг 5. Открой Terminal в cPanel

Перейди в:

```bash
cd ~/apps/botfactory
```

И выполни:

```bash
npm ci
npm run build
mkdir -p tmp
```

Если `npm ci` вдруг не сработает, попробуй:

```bash
npm install
npm run build
mkdir -p tmp
```

Но в норме лучше именно `npm ci`.

## Шаг 6. Добавь переменную окружения

В настройках Node.js приложения добавь:

```text
NODE_ENV=production
```

Обычно этого достаточно.

Отдельно `PORT` руками задавать чаще всего не нужно. cPanel/Passenger обычно передаёт его сам.

## Шаг 7. Запусти или перезапусти приложение

В интерфейсе cPanel нажми:

- `Start App`
- или `Restart`

Если приложение уже было запущено и ты обновил код, дополнительно можно выполнить:

```bash
touch ~/apps/botfactory/tmp/restart.txt
```

Это часто помогает Passenger понять, что приложение надо перечитать.

## Проверка после деплоя

После запуска обязательно открой в браузере:

- `/`
- `/services`
- `/services/web-development-minsk`
- `/robots.txt`
- `/sitemap.xml`
- `/opengraph-image`
- `/services/ai-integration-minsk/opengraph-image`

### Что должно быть

- главная страница открывается без `500`
- стили загрузились
- 3D/анимации не ломают страницу
- `robots.txt` открывается
- `sitemap.xml` открывается
- Open Graph image маршруты не возвращают ошибку

## Что важно именно для этого проекта

### 1. Проект не статический

Здесь используется полноценный Next.js runtime. Просто выгрузить файлы в `public_html` недостаточно.

### 2. Главная страница довольно тяжёлая

В сборке главная страница имеет заметный клиентский JS bundle. Это не блокер для деплоя, но на слабом хостинге и слабых устройствах нужно после запуска проверить:

- скорость первой загрузки
- отсутствие лагов
- корректную работу мобильной версии

### 3. Есть динамические social image маршруты

Это маршруты вида:

- `/opengraph-image`
- `/twitter-image`
- `/services/[slug]/opengraph-image`
- `/services/[slug]/twitter-image`

Они локально работают, но после деплоя их всё равно надо проверить отдельно.

### 4. В проекте используются Google Fonts через `next/font/google`

Во время `npm run build` сервер скачивает нужные шрифты.

Если build на сервере падает на шрифтах, причина обычно одна из двух:

- сервер не имеет исходящего доступа в интернет
- сервер не может достучаться до Google

Тогда есть 2 пути:

1. решать это с хостингом
2. переводить проект на локальные шрифты через `next/font/local`

## Частые ошибки и что делать

## Ошибка: нет пункта `Setup Node.js App`

Причина:

- тариф cPanel не поддерживает Node.js приложения

Что делать:

- писать в поддержку хостинга
- переходить на другой тариф
- переносить проект на VPS/Vercel

## Ошибка: Node.js только 16 или 18

Причина:

- у хостинга слишком старая версия Node.js

Что делать:

- попросить Node.js `20+`
- если нельзя, лучше не мучить этот хостинг

## Ошибка: `npm run build` падает

Что проверить:

- точно ли выбрана правильная версия Node.js
- выполнен ли `npm ci`
- есть ли доступ сервера в интернет
- хватает ли памяти процессу сборки

Сначала просто повтори:

```bash
npm ci
npm run build
```

## Ошибка: `npm: command not found`

Причина почти всегда одна из двух:

- Node.js окружение в cPanel вообще не подключено
- на тарифе нет нормальной поддержки Node.js приложений

Что это значит на практике:

- просто открыть `Terminal` недостаточно
- сначала обычно нужно создать приложение в `Setup Node.js App` / `Application Manager`
- после этого cPanel создаёт Node.js окружение и даёт команду активации

Что делать по шагам:

1. В cPanel в поиске сверху введи `node`.
2. Найди `Setup Node.js App` или `Application Manager`.
3. Если такого пункта нет, значит на этом тарифе Node.js, скорее всего, не доступен.
4. Если пункт есть, сначала создай Node.js приложение.
5. После создания приложения открой его карточку и найди команду активации окружения.
6. Выполни эту команду в Terminal.

Обычно она выглядит примерно так:

```bash
source /home/USERNAME/nodevenv/apps/botfactory/20/bin/activate
```

После этого проверь:

```bash
node -v
npm -v
```

И только потом выполняй:

```bash
cd ~/apps/botfactory
npm ci
npm run build
```

Если после создания Node.js приложения `npm` всё равно не найден:

- пришли в поддержку хостинга скрин и текст `npm: command not found`
- попроси включить доступ к Node.js / npm в shell для cPanel Node.js app

## Ошибка: сайт открывается, но `500 Internal Server Error`

Что делать:

1. проверить, выполнилась ли сборка без ошибок
2. проверить, что стартовый файл в cPanel указан как `app.js`
3. выполнить:

```bash
cd ~/apps/botfactory
mkdir -p tmp
touch tmp/restart.txt
```

4. перезапустить приложение
5. посмотреть логи в cPanel

## Ошибка: не открывается `/opengraph-image`

Что делать:

1. проверить, что основной сайт вообще работает
2. проверить логи приложения
3. проверить, что сборка завершилась до конца

Если падают только social image маршруты, а остальной сайт жив, значит проблема почти наверняка в runtime этих маршрутов или в окружении хостинга.

## Ошибка: после загрузки архива ничего не работает

Частая причина:

- проект загрузили, но не выполнили `npm ci`
- проект загрузили, но не выполнили `npm run build`
- приложение не подключено через `Setup Node.js App`

Загрузка файлов сама по себе не запускает Next.js проект.
